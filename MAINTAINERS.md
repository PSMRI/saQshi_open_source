## Release Approval Rules

A public release should not be published until these approvals are recorded:

1. Product maintainer approval.
2. Technical maintainer approval.
3. Security contact approval.
4. Release manager approval.
5. Data owner approval for public master/configuration data.
6. Legal/compliance approval for license, privacy and redistribution.

## Issue Handling Rules

- Security issues must be reported privately through the official security contact.
- Public issues should not include credentials, database dumps, uploads, screenshots with personal data, facility-sensitive data or exploit details.
- Issue triage should classify each issue as bug, security, documentation, enhancement, data/configuration or deployment.
- Security issues should follow `SECURITY.md`.
